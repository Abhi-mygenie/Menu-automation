### Step 2 вЂ” CR-MENU-0001: Pre-Export Correction Diff View

**What the user asked for:**
> "During the export, do I see the corrected UI with any color marker just to identify once more quickly before getting into the POS?"

**What was built:**
The Export Modal now shows a **"CORRECTIONS TO REVIEW"** section between the Review Summary counts and the Export button. It lists every non-approve correction color-coded by action type:

| Action | Color | What shows |
|---|---|---|
| EDITED | Blue | Original name (strikethrough) в†’ corrected name; rate if changed |
| DELETED | Red | Item name in red strikethrough |
| UNCLEAR | Amber | Item name + warning: "Will be exported with original AI values вЂ” verify before POS push" |
| ADDED | Purple | Corrected name (new row added by reviewer) |

If all rows were approved: shows a green "All reviewed rows and notes were approved вЂ” nothing to inspect" message instead.

**Files changed:**
- `frontend/src/components/review/ExportModal.jsx` вЂ” full replacement (added: ACTION_CONFIG, buildDiffItems(), DiffItemRow, CorrectionsDiffSection components; updated: function signature, useMemo, JSX)
- `frontend/src/pages/review/ReviewWorkspace.jsx` вЂ” 3 changes: handleAddRow fix + 2 new props to ExportModal

**Gap discovered and fixed:** `handleAddRow` in ReviewWorkspace never pushed `add_missing` rows into `corrections` state. This was fixed by adding a synthetic key (`"pageNo-am-timestamp"`) push after the API call succeeds.

**Formal documents created:**
